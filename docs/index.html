A processor of textual tree data structures
A software artifact generator

Produce artifacts processing source streams of indented text

Wizzi allow you to merge and manipulate textual trees of data.
Sources can be schemed and when loaded may expose functions on data.
You can merge source and context data.
Artifact productions can be driven by data.

  • Sources

    Sources are editable streams of Ittf (Indented Text Tree Format) documents, a tree data structure where nodes are lines of name/value pairs.

  • Templates

    Ittf documents are templates, can be merged with context data, can be scripted and can include and merge other Ittf documents.

    The Ittf processor is a script and template engine that transforms textual source streams into mTree memory objects.

  • Schemas

    Domain specific schemas can be applied to mTrees, creating dynamic context objects for data merging and generations.

  • Automate generations

    Generations can be automated chaining tasks in Wizzi Jobs.

Main components

Wizzi is a javascript Api that can run localy, on NodeJs servers or in the browser.

It is language agnostic.

User created schemas, transformers and generators target domain specific artifacts.

  • Ittf document

    Wizzi source document.

    Represents code and data as a textual indented tree data structure.

    Composable, templeatable, scriptable.

    more

  • Wizzi schema

    Formal description of a type of Ittf document, similar to XML Schema.

    Drives the generation of a Wizzi Model Dom that load and validate the Ittf document of its type.

    more

  • Wizzi model

    Instance of a Wizzi Model Dom.

    When loaded with a Ittf document can be used as a context object in generations.

    more

  • Model transformer

    Transforms a Wizzi Model.

    more

  • Artifact generator

    Generates a software artifact merging data from Wizzi models.

    more

  • Wizzi factory

    The main API component of Wizzi.

    Loads plugins, processes sources, execute generations, run jobs.

    more

  • Wizzi job

    Sequence of generation tasks run by the Wizzi factory.

    more

  • Wizzi plugin

    Implementation of domain specific schemas, transformers and generators.

A pluggable architecture

Wizzi is made of an ecosystem of domain specific plugins.

Plugins are the horse work of generations.

Core and default plugins support the creation of user generated plugins.

  • Core plugins

    The ` wizzi-core ` plugin contains schemas and artifacts used by the kernel modules.

    more

  • Default plugins

    Default plugins, available in the Npm registry, can be used as starters: the `wizzi-js `, the ` wizzi-web ` and the ` wizzi-lab ` plugin.

    The `wizzi-js` plugin is used to generate Wizzi itself.

    more

  • Npm plugins

    Plugin authors can contribute their plugins publishing them to the Npm registry.

    Public Wizzi plugins should be prefixed with 'wizzi-plugin-'.

    more

  • Local plugins

    User generated plugins can be declared by path to the Wizzi factory and may remain locals to user projects.

    more

Developing in a connected world

Wizzi is designed to operate in the browser and to use cloud Git repositories as data sources and destinations.

Web Hooks an CI Tools may integrate Wizzi in a cloud development environment.

  • The Browser is the IDE

    Wizzi adopts BrowserFS to use the browser as a file system.

  • Git is the code DB

    Wizzi uses isomorphic-git, an in-browser Git API, to manage source documents in the cloud.

  • Web Hooks and CI

    Generated artifacts commited to a Git Repository can be immediately integrated thanks to Web Hooks and Continuous integration tools.
